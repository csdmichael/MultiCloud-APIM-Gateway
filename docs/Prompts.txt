# Architecture

## Identity Flow
AWS IAM Identity Center (AWS SSO) is already federated with Microsoft Entra ID.

Authentication flow:
1. User authenticates via AWS SSO
2. Microsoft Entra ID issues JWT token
3. Client calls APIM endpoint with Bearer token
4. APIM validates JWT
5. APIM extracts user + group claims
6. APIM applies LLM token governance
7. APIM returns mocked JSON response

- AWS Bedrock

# JWT Validation Requirements

Implement APIM policy using:
validate-azure-ad-token

The token validation must verify:
- issuer
- audience
- tenant
- expiration

Store validated JWT in output variable.

Use placeholders:
- tenantId
- apimAudience
- apimClientAppId

Example direction:

<validate-azure-ad-token
    tenant-id="{{tenantId}}"
    header-name="Authorization"
    failed-validation-httpcode="401"
    output-token-variable-name="jwt">

    <audiences>
        <audience>{{apimAudience}}</audience>
    </audiences>

    <client-application-ids>
        <application-id>{{apimClientAppId}}</application-id>
    </client-application-ids>

</validate-azure-ad-token>
